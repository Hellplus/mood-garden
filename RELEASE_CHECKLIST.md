# Mood Garden 发布检查清单

每次发布或更新 GitHub Pages 前，可以按下面清单逐项检查。

## 页面访问

- [ ] GitHub Pages 地址可以正常打开。
- [ ] 页面没有明显加载错误或空白。
- [ ] 浏览器标题显示清楚。
- [ ] 首页 Hero 区域显示正常。
- [ ] Hero 动画不过度、不影响阅读。

## 主题与视觉

- [ ] 浅色主题显示正常。
- [ ] 深色主题显示正常。
- [ ] 治愈粉主题显示正常。
- [ ] 主题切换后页面样式立即更新。
- [ ] 刷新页面后主题偏好可以恢复。
- [ ] 图标动画显示正常。
- [ ] 移动端布局不拥挤、不溢出。

## 情绪记录功能

- [ ] 可以选择情绪。
- [ ] 可以输入一句心情。
- [ ] 点击“种下一朵花”后可以生成花卡片。
- [ ] 空输入时不会提交，并给出提示。
- [ ] 随机花语根据情绪生成。
- [ ] `moodIcon` 可以正常生成并保存。
- [ ] 刷新页面后花卡片和 `moodIcon` 保持不变。
- [ ] Hero 动态图标可以正常随机显示。
- [ ] Hero 动态文案可以正常随机显示。
- [ ] 种花后 Hero 图标和文案会更新为对应情绪。

## 浏览管理

- [ ] 按情绪筛选正常。
- [ ] 按关键词搜索正常。
- [ ] 最新优先排序正常。
- [ ] 最早优先排序正常。
- [ ] 重置筛选正常。
- [ ] 当前显示数量正确。
- [ ] 筛选、搜索、排序不会改变 `localStorage` 原始数据。

## 编辑与删除

- [ ] 可以编辑单张花卡片的心情文字。
- [ ] 编辑为空时不会保存，并给出提示。
- [ ] 编辑保存后页面立即更新。
- [ ] 刷新后编辑内容仍然存在。
- [ ] 可以删除单张花卡片。
- [ ] 删除前有二次确认。
- [ ] 删除后页面、统计和导出数据同步更新。

## 统计与分析

- [ ] 情绪统计总数正确。
- [ ] 每种情绪数量正确。
- [ ] 最近 7 天记录统计正常。
- [ ] 情绪分布统计正常。
- [ ] 最常出现的情绪显示正常。
- [ ] 连续记录天数显示正常。
- [ ] 本周情绪回顾文案显示正常。
- [ ] 数据分析基于全部记录，不受筛选、搜索、排序影响。
- [ ] 数据不足时显示温柔空状态。

## 导出与导入

- [ ] txt 情绪日记可以正常导出。
- [ ] txt 导出内容包含日期、情绪、心情和随机花语。
- [ ] JSON 备份可以正常导出。
- [ ] JSON 备份包含 app、version、exportedAt 和 records。
- [ ] JSON 备份可以正常导入。
- [ ] 导入前可以预览记录数量和情绪分布。
- [ ] 合并导入正常。
- [ ] 覆盖导入正常，并有二次确认。
- [ ] 错误格式的 JSON 会给出友好提示。
- [ ] 导入后花卡片、统计、分析和筛选结果会刷新。

## 清空与本地存储

- [ ] 清空花园前有二次确认。
- [ ] 清空后页面花卡片清空。
- [ ] 清空后情绪统计归零。
- [ ] 清空后数据分析显示空状态。
- [ ] `localStorage` 刷新恢复正常。
- [ ] 更换浏览器或清理浏览器数据后，不会误以为数据仍然存在。

## 文档与发布资料

- [ ] README 包含线上访问地址。
- [ ] README 包含项目截图。
- [ ] README 包含主要功能说明。
- [ ] README 包含本地运行方式。
- [ ] README 包含数据保存说明。
- [ ] README 包含注意事项。
- [ ] CHANGELOG 记录 V1.0 到当前版本的主要变化。
- [ ] PROJECT_BRIEF 适合作为作品集项目说明。

## 发布风险

- [ ] 项目中没有隐私数据。
- [ ] 项目中没有本地绝对路径。
- [ ] 项目中没有无效外部资源。
- [ ] 项目没有引入未说明的第三方库。
- [ ] GitHub Pages 使用的资源路径都是相对路径。
- [ ] 本项目明确说明只用于轻量情绪记录。
- [ ] 本项目明确说明不提供心理诊断或医疗建议。
