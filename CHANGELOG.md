# 更新记录

这个文件记录 Mood Garden 情绪花园的主要版本变化。项目保持 HTML、CSS、JavaScript 三文件主体结构，数据保存在浏览器 `localStorage` 中。

## V2.2 记录详情与日记增强

- 增加花卡片详情弹窗，可以查看情绪、图标、日期、心情文字、花语、强度、标签、收藏状态和补充日记。
- 种花时可以选择 1-5 级心情强度，并输入最多 3 个今日关键词。
- 增加收藏这一朵功能，收藏状态保存为单条记录的可选字段。
- 详情中可以编辑心情文字、强度、标签、收藏状态和补充日记，不修改情绪、图标、日期和花语。
- txt 日记导出补充强度、标签、收藏状态和补充日记。
- JSON 备份导出会保留新字段，导入旧备份时缺少新字段也能兼容。
- 日历当天记录支持打开详情，数据分析仍按情绪和日期运行。
- 保持原有 `localStorage` 主结构不变，旧记录可以继续正常显示。

## V2.1 日历视图升级

- 在花园区域新增列表 / 日历视图切换，默认仍保留原有列表视图。
- 新增当前月日历，支持上一月、下一月和回到今天。
- 有记录的日期会显示数量和少量情绪图标，避免日历格子过于拥挤。
- 点击某一天后，可以在日历下方查看当天记录。
- 日历基于全部花卡片记录生成，不受当前筛选、搜索和排序影响。
- 保持原有 `localStorage` 主结构不变。

## V2.0 产品化升级

- 新增 `manifest.webmanifest`，补充应用名称、启动路径、显示模式、主题色和图标配置。
- 新增 `service-worker.js`，缓存 `index.html`、`style.css`、`script.js`、manifest 和应用图标，提供基础离线访问能力。
- 新增 `assets/icon.svg`，用于 PWA 图标和浏览器图标。
- 增加首次使用引导，说明选择情绪、写一句心情、种下一朵花和本地保存方式。
- 补充本地数据说明：记录只保存在当前浏览器，建议使用 JSON 备份长期保存。
- 将页面版本号升级为 `v2.0`，并同步更新 README。

## V1.8 移动端信息架构优化

- 手机端新增记录 / 花园 / 分析 / 数据四个区域。
- 手机端使用底部导航降低长页面的信息拥挤感。
- 将筛选、搜索、排序放入花园区，将数据分析放入分析区，将导入导出和说明放入数据区。
- 桌面端继续保留完整展示布局。
- 修复移动端记录区 Today 动态卡片缺失问题。

## V1.7 最终展示完善

- 优化 README 为更适合 GitHub 首页展示的最终版。
- 新增项目总结、发布检查清单、Codex 学习复盘和提示词模板等文档。
- 补充项目截图引用，方便作品集或仓库首页展示。

## V1.6 发布质量整理

- 优化浏览器标题、页面描述和基础分享信息。
- 增加页面版本号、轻量使用说明和底部发布信息。
- 补充 README 展示内容，并新增 CHANGELOG 版本记录。
- 明确说明数据保存在当前浏览器中，清空花园会删除当前浏览器里的记录。

## V1.5 成品化与代码整理

- 整理情绪配置、花语池、图标池、Hero 文案池和动画映射。
- 统一 `localStorage` 读取、保存和清空逻辑。
- 整理页面刷新流程，让花卡片、统计、筛选和分析更容易同步。
- 补强基础可访问性和错误处理。

## V1.4 数据分析与动态图标

- 增加最近 7 天记录统计、情绪分布、最常出现情绪和连续记录天数。
- 增加本周情绪回顾文案，基于用户记录做轻量总结。
- 增加情绪动态图标池、图标专属动画和 Hero 动态文案。
- 情绪回顾仅用于个人记录，不提供心理诊断或医疗建议。

## V1.3 视觉体验增强

- 增加浅色、深色、治愈粉三种主题，并保存主题偏好。
- 增加 Toast 轻提示、空状态和无结果状态。
- 优化页面布局、Hero 区、花卡片样式、响应式布局和交互动画。
- 支持 `prefers-reduced-motion`，照顾减少动态效果的系统设置。

## V1.2 JSON 备份导入导出

- 增加 JSON 备份导出功能。
- 增加 JSON 备份导入功能，支持导入前预览。
- 支持覆盖导入和合并导入。
- 增加导入格式错误提示和旧数据兼容处理。

## V1.1 浏览管理增强

- 增加一键重置筛选和当前显示结果数量。

## V1.0 基础情绪记录

- 创建基础页面结构：标题、情绪选择、心情输入框和种花按钮。
