# 更新记录

这个文件记录 Mood Garden 情绪花园的主要版本变化。项目保持 HTML、CSS、JavaScript 三文件主体结构，数据保存在浏览器 `localStorage` 中。

## V2.6 真实使用体验优化

- 在记录区增加今日记录状态卡，显示今天是否已经记录、今日记录数量和最近一次情绪。
- 在心情输入框附近增加随机记录引导问题，并支持换一个提示。
- 种花成功后增加下一步引导，可以去花园看看，也可以继续记录。
- 在记录区增加最近 1-3 条记录快速回看，点击可以打开详情。
- 今日状态和最近记录基于全部本地记录计算，不受当前筛选、搜索、标签、收藏或强度筛选影响。
- 不新增通知权限，不做心理诊断，也不改变 `localStorage` 主结构。

## V2.5 数据洞察体验优化

- 将数据分析区整理为更清晰的情绪洞察仪表盘。
- 增加洞察总览，集中展示总记录数、本月记录数、常见情绪、平均强度和常用标签。
- 增加今日 / 本周 / 本月内部切换，让移动端分析 tab 不再一次性堆叠全部卡片。
- 今日洞察展示今日记录数、今日情绪和今日强度，不做状态判断。
- 本周洞察整合最近 7 天统计、本周常见情绪、本周常用标签、连续记录和本周回顾。
- 本月洞察整合最近 30 天趋势、本月情绪分布、本月强度、标签、收藏回顾和月度回顾。
- 分析仍然基于保存的记录数据，不受当前筛选、搜索、排序、标签筛选、收藏筛选和强度筛选影响。

## V2.4 数据分析进阶

- 增加最近 30 天记录趋势，用轻量 CSS 条形图展示每天记录数量。
- 增加心情强度分析，统计已设置强度记录的平均强度、最高强度数量和低强度数量。
- 增加常用标签 Top 5，基于全部记录中的 `tags` 统计。
- 增加收藏记录回顾，显示收藏总数和最近收藏的记录摘要。
- 增加情绪 × 强度关联分析，按情绪统计平均强度，只做轻量回顾，不提供心理诊断。
- 增加月度回顾摘要，基于当前月份记录生成规则化总结文案。
- V2.4 分析仍然基于全部记录，不受筛选、搜索、排序、标签筛选、收藏筛选和强度筛选影响。
- 旧记录没有 `intensity`、`tags` 或 `isFavorite` 时继续兼容显示。

## V2.3 标签与收藏管理

- 增加只看收藏筛选，可以快速查看已收藏的花卡片。
- 根据全部记录中的 `tags` 生成标签云，标签显示名称和数量。
- 点击标签后只显示包含该标签的记录，再次点击当前标签可以取消标签筛选。
- 增加心情强度筛选，支持按 1-5 级强度浏览记录。
- 情绪、关键词、收藏、标签、强度和排序可以组合使用。
- 增加当前筛选状态摘要，并增强一键重置筛选。
- 标签、收藏和强度筛选只影响当前显示，不改变原始数据；日历、数据分析和导出仍基于全部记录。

## V2.2 记录详情与日记增强

- 增加花卡片详情弹窗，可以查看情绪、图标、日期、心情文字、花语、强度、标签、收藏状态和补充日记。
- 种花时可以选择 1-5 级心情强度，并输入最多 3 个今日关键词。
- 增加收藏这一朵功能，收藏状态保存为单条记录的可选字段。
- 详情中可以编辑心情文字、强度、标签、收藏状态和补充日记，不修改情绪、图标、日期和花语。
- txt 日记导出补充强度、标签、收藏状态和补充日记。
- JSON 备份导出会保留新字段，导入旧备份时缺少新字段也能兼容。
- 日历当天记录支持打开详情，数据分析仍按情绪和日期运行。
- 保持原有 `localStorage` 主结构不变，旧记录可以继续正常显示。

## V2.1 日历视图升级

- 在花园区域新增列表 / 日历视图切换，默认仍保留原有列表视图。
- 新增当前月日历，支持上一月、下一月和回到今天。
- 有记录的日期会显示数量和少量情绪图标，避免日历格子过于拥挤。
- 点击某一天后，可以在日历下方查看当天记录。
- 日历基于全部花卡片记录生成，不受当前筛选、搜索和排序影响。
- 保持原有 `localStorage` 主结构不变。

## V2.0 产品化升级

- 新增 `manifest.webmanifest`，补充应用名称、启动路径、显示模式、主题色和图标配置。
- 新增 `service-worker.js`，缓存 `index.html`、`style.css`、`script.js`、manifest 和应用图标，提供基础离线访问能力。
- 新增 `assets/icon.svg`，用于 PWA 图标和浏览器图标。
- 增加首次使用引导，说明选择情绪、写一句心情、种下一朵花和本地保存方式。
- 补充本地数据说明：记录只保存在当前浏览器，建议使用 JSON 备份长期保存。
- 将页面版本号升级为 `v2.0`，并同步更新 README。
