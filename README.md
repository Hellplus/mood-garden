# Mood Garden 情绪花园

一个用 HTML、CSS 和 JavaScript 制作的小型前端学习作品。用户可以记录当天的情绪和一句心情，种下一张属于自己的情绪花卡片。

在线访问地址：[https://hellplus.github.io/mood-garden/](https://hellplus.github.io/mood-garden/)

## 项目简介

Mood Garden 情绪花园是一个偏温柔、治愈风格的私人情绪日记页面。页面已优化为治愈系情绪日记风格，支持浅色、深色、治愈粉三种主题、响应式布局和更完整的视觉体验。项目已完成成品化与发布质量整理，在保持 HTML、CSS、JavaScript 三文件结构的基础上，整理了配置、`localStorage` 读写、渲染流程、主题样式、动画系统、页面说明和版本记录。它不依赖后端，也不使用框架，数据保存在浏览器的 `localStorage` 中。

项目中的情绪回顾只基于用户自己记录的内容做轻量总结，不提供心理诊断或医疗建议。

这个项目主要适合前端新手练习：

- HTML 页面结构
- CSS 布局和视觉样式
- JavaScript DOM 操作
- 浏览器本地存储 `localStorage`
- 简单的数据渲染和状态更新

- 每个情绪图标拥有对应的专属动画
- 云朵、星星、气球、叶子、花朵、月亮等图标会呈现不同动态效果
- Hero 图标、花卡片图标和部分情绪统计图标复用统一动画系统
- 随机生成对应情绪的今日花语
- 花卡片保存到 `localStorage`
- 刷新页面后仍能看到历史花卡片
- 显示情绪统计小面板
- 显示最近 7 天记录统计
- 显示情绪分布统计
- 显示最常出现的情绪
- 显示连续记录天数
- 根据最近记录生成本周情绪回顾
- 数据不足时显示温柔提示
- 支持按情绪筛选花卡片
- 支持按关键词搜索心情文字
- 支持最新优先 / 最早优先排序
- 支持一键重置筛选条件
- 显示当前筛选结果数量
- 支持导出情绪日记为 `.txt` 文本文件
- 支持导出 JSON 备份文件
- 支持导入 JSON 备份文件
- 导入前会预览记录数量和情绪分布
- 支持覆盖导入和合并导入
- 导入格式错误时会给出提示
- 支持编辑单条情绪记录的心情文字
- 支持删除单条情绪记录，删除前会二次确认
- 支持清空花园，清空前会二次确认
- 支持浅色 / 深色 / 治愈粉主题切换
- 支持保存主题偏好，刷新后保持上次选择
- 使用 Toast 轻提示展示操作反馈
- 增加页面进入动画、Hero 头部呼吸感动效、花卡片交互动画、Toast 出入场动画、主题切换过渡和移动端交互优化
- 区分空花园状态和筛选无结果状态
- 针对手机端做了简单的响应式体验优化
- 页面包含轻量使用说明、项目版本号和底部发布信息
- README 和 CHANGELOG 记录项目功能与版本变化

## 本地运行方式

这个项目可以直接在浏览器中运行：

1. 下载或克隆项目到本地。
2. 找到项目目录中的 `index.html`。
3. 双击 `index.html`，用浏览器打开即可。

也可以使用 VS Code 的 Live Server 插件打开，方便开发时自动刷新。

使用页面时，可以选择情绪、写下一句心情，然后点击“种下一朵花”保存记录。历史记录可以通过筛选、搜索和排序整理。如果想保存一份文字版日记，可以点击“导出日记”按钮，下载包含历史花卡片的 `.txt` 文件。也可以点击“导出备份”保存 JSON 数据文件，之后通过“导入备份”恢复或合并记录。

所有花卡片数据都保存在当前浏览器的 `localStorage` 中。清空花园会删除当前浏览器里的记录，导入备份时可以选择合并或覆盖当前记录。

## 项目文件结构

```text
mood-garden/
├─ index.html
├─ style.css
├─ script.js
├─ CHANGELOG.md
└─ README.md
```

文件说明：

- `index.html`：页面结构，包括标题、情绪选择、输入框、统计面板和花园区域。
- `style.css`：页面样式，包括背景、按钮、卡片、统计面板和响应式布局。
- `script.js`：页面交互逻辑，包括种花、随机花语、保存数据、读取历史、情绪统计、筛选搜索、导入导出和清空花园。
- `CHANGELOG.md`：版本更新记录，整理从 V1.0 到 V1.6 的主要变化。
- `README.md`：项目说明文档。
