# Mood Garden 情绪花园项目说明

## 项目名称

Mood Garden 情绪花园

## 项目定位

Mood Garden 情绪花园是一个使用 HTML、CSS 和 JavaScript 从零完成的轻量情绪记录前端小作品。它把每天的情绪和一句心情保存成一张花卡片，让用户可以用更温柔、可视化的方式回看自己的日常记录。

这个项目适合作为个人作品集中的前端练习项目，重点展示原生前端开发、页面交互、本地存储、数据渲染、响应式布局和 GitHub Pages 发布能力。

## 为什么做这个项目

我希望做一个不依赖后端、不使用框架，但功能相对完整的小型前端项目。相比普通待办清单或表单练习，情绪花园这个主题更适合练习 UI 氛围、状态反馈、动画、数据统计和用户体验细节。

项目目标不是做一个商业级产品，而是通过一个完整的小作品，练习从需求拆解、功能迭代、视觉优化、代码整理到发布展示的完整流程。

## 核心功能

- 选择每日情绪：开心、平静、焦虑、疲惫、兴奋
- 写下一句今天的心情
- 生成情绪花卡片
- 根据情绪生成随机今日花语
- 根据情绪随机生成图标，并保存到花卡片中
- 花卡片、Hero 图标和部分统计图标带有轻量动画
- 使用 `localStorage` 保存历史记录，刷新后自动恢复
- 编辑或删除单条花卡片
- 按情绪筛选、按关键词搜索、按时间排序
- 导出 `.txt` 情绪日记
- 导出和导入 JSON 备份
- 显示情绪统计、最近 7 天记录、情绪分布、最常出现情绪、连续记录天数和本周回顾
- 支持浅色、深色、治愈粉三种主题
- 使用 Toast 提示操作结果
- 支持移动端响应式布局
- 手机端通过记录 / 花园 / 分析 / 数据四个区域提升 App 感
- 支持 PWA 基础配置，包括 manifest、应用图标和 service worker
- 支持基础离线访问，核心页面资源可以被缓存
- 首次打开时显示简短使用引导
- 在页面中说明数据只保存在当前浏览器，并建议使用 JSON 备份

## 技术栈

- HTML
- CSS
- JavaScript
- localStorage
- PWA Manifest
- Service Worker
- GitHub Pages

项目没有使用 React、Vue、构建工具、后端服务或第三方图表库。

## 数据如何保存

项目数据保存在浏览器的 `localStorage` 中。

花卡片数据以数组形式保存，每条记录包含情绪、心情文字、创建日期、随机花语、图标、唯一 id 等信息。页面打开时会读取本地数据并渲染历史花卡片；新增、编辑、删除、导入或清空后，会同步更新 `localStorage` 和页面视图。

因为数据保存在当前浏览器中，所以更换浏览器、清理浏览器数据或使用无痕模式时，可能看不到原来的记录。项目提供 JSON 备份导入导出功能，用于手动保存和恢复数据。

V2.0 中，页面进一步补充了本地保存说明和首次使用引导，提醒用户记录只保存在当前浏览器中。PWA 离线能力只缓存页面静态资源，不会把数据同步到云端，也不会改变原有花卡片数据结构。

## 我如何使用 Codex 逐步完成项目

这个项目是通过和 Codex 进行多轮协作逐步完成的。我先从最小可运行版本开始，只实现标题、情绪选择、输入框、种花按钮和本地保存。之后每次围绕一个明确目标迭代，例如增加随机花语、统计面板、编辑删除、导入导出、主题切换、数据分析、动画系统和发布文档。

在每个阶段，我通常先让 Codex 分析当前结构和修改方案，确认后再执行小范围代码修改。修改完成后，再进行功能验收、问题修复和 README 更新。这样的方式让项目逐步扩展，同时尽量避免一次性大改带来的风险。

## 项目亮点

- 使用原生 HTML、CSS、JavaScript 完成，结构适合新手阅读
- 功能从基础记录逐步迭代到筛选、导入导出、数据分析和主题系统
- 使用 `localStorage` 完成无后端的数据保存
- 支持 JSON 备份导入导出，降低本地数据丢失风险
- 视觉风格统一，包含主题切换、轻量动画和响应式布局
- 数据分析基于本地记录生成，不依赖外部服务
- 保留完整 README、CHANGELOG 和项目说明，适合公开展示
- 增加基础 PWA 能力，让项目更接近可安装、可离线打开的轻量 App 形态

## 遇到的挑战

- 如何在不使用框架的情况下管理不断增加的页面状态
- 如何让新增、编辑、删除、导入、清空后，花卡片、统计、筛选和分析同步更新
- 如何兼容旧数据，例如早期记录没有 `id`、`flowerQuote` 或 `moodIcon`
- 如何让动画有生命感，但不过度影响阅读和性能
- 如何控制功能增长，避免项目从新手作品变成难以维护的大工程
- 如何让 README 和项目说明真实清楚，而不是写成夸张的商业产品介绍

## 学到的内容

- 如何用 HTML、CSS、JavaScript 搭建完整小型前端项目
- 如何用 `localStorage` 保存和恢复本地数据
- 如何设计数据结构并兼容旧版本数据
- 如何用 DOM 操作渲染列表、统计、筛选结果和空状态
- 如何实现导出 txt、导出 JSON 和导入 JSON
- 如何用 CSS 变量管理主题
- 如何用 `keyframes` 制作轻量动画，并支持 `prefers-reduced-motion`

- 如何通过统一刷新流程降低重复逻辑
- 如何为 GitHub Pages 项目准备 README、CHANGELOG、截图和项目说明
- 如何配置 manifest 和 service worker，并兼顾 GitHub Pages 的 `/mood-garden/` 路径
